Understanding CSS Structure

A well-organized stylesheet is easier to maintain and modify. Each section should have clear comments explaining its purpose.

Key Principles:

1. Group related styles together

2. Use descriptive section comments

3. Maintain consistent indentation

4. Use meaningful class and ID names

CSS Syntax Rules

Every CSS rule consists of a selector and a declaration block:

selector { property: value; property: value; }

Example:

/* This is a comment - it spans multiple lines and helps explain the purpose of the styles */ header { background-color: #2c3e50; color: white; padding: 20px; }

Writing Effective Comments

Comments in CSS start with /* and end with */. Use them to:

1. Label major sections with section headers

2. Explain complex selectors or rules

3. Mark TODO items for future updates

4. Group related properties

/* ========================================== NAVIGATION STYLES ========================================== */ nav { background-color: #34495e; padding: 15px; } /* Style for main CTA button - Update colors quarterly */ .cta-button { background-color: #007BFF; color: white; padding: 12px 24px; border-radius: 5px; }

Organizing Your Stylesheet

Follow this structure for larger projects:

1. Reset/Normalize styles

2. Global/Base styles

3. Typography styles

4. Layout/Component styles

5. Page-specific styles

6. Utility classes

7. Media queries
