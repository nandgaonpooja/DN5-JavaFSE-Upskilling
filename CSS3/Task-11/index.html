Debug and Test with DevTools

1. Inspect Element Basics

Right-click on any element and select "Inspect" to open Chrome DevTools and examine the HTML/CSS.

Right-click me and select "Inspect" to see the HTML structure

In the Elements panel, you can:

  • View the HTML hierarchy
  • See applied CSS styles
  • Inspect computed styles
  • Modify HTML/CSS in real-time

2. View Applied CSS

When you inspect an element, the Styles panel shows all CSS affecting it.

Inspector Panel Features:
- View inline styles
- See stylesheet sources
- Check specificity
- See overridden rules (struck through)
- Toggle rules on/off
Inspect this box to see all CSS rules affecting it

3. Edit Styles in Real-Time

Click on any CSS property in DevTools to edit it instantly and see changes live.

4. Console Tab

The Console is where JavaScript errors appear and where you can run commands.

Common Console shortcuts:
- document.querySelector() - Find elements
- console.log() - Print messages
- $0 - References selected element
- $1, $2, $3 - Previous selections
Open the Console tab (F12 > Console) and click the button above

Step Over Execute one line Press F10 or click Step Over
Step Into Enter function calls Press F11 or click Step Into
Step Out Exit function Shift+F11 or click Step Out
Variables View variable values Check Scope panel during debug

7. Network Tab

Monitor all network requests made by the page.

Network Tab shows:
- Request method (GET, POST, etc.)
- Status codes (200, 404, etc.)
- File size and transfer size
- Time taken
- Response headers and body

8. Performance Tab

Analyze page load time and runtime performance.

Performance Tab helps:
1. Record page interactions
2. Identify slow operations
3. Optimize rendering performance
4. Find memory leaks
5. Improve user experience

9. Device Mode (Responsive Testing)

Test your site on different devices and screen sizes.

How to Open Device Mode:
- Keyboard: Ctrl+Shift+M (Windows) or Cmd+Shift+M (Mac)
- Menu: DevTools > More tools > Device Mode
In Device Mode:
- Select preset devices (iPhone, iPad, etc.)
- Test different screen sizes
- Toggle orientation (portrait/landscape)
- Simulate network throttling
- Test touch events

11. Testing Forms in DevTools

Test form elements with focus states:

input[type="text"]:focus { outline: 3px solid #3498db; border-color: #3498db; }

12. Best Practices

  • Test on real devices: Emulation is helpful but not perfect
  • Check all breakpoints: Test mobile, tablet, and desktop
  • Monitor performance: Aim for fast load times and smooth interactions
  • Validate HTML/CSS: Use validators to catch errors
  • Test across browsers: Use different browsers to ensure compatibility
  • Check accessibility: Use Lighthouse and accessibility tools
  • Inspect network: Optimize asset sizes and reduce requests
  • Clear cache: Disable cache while developing for accurate testing
